A **directive** — one sentence of intent — becomes a **mission**. `create_mission`
takes the directive and either a custom plan or, when omitted,
`mission.DefaultPlan(directive)`: research → design → build-core → build →
test ∥ secops ∥ perf → integrate → docs → retro. The plan's phases are
persisted and turned into an executable task queue that agents pull from.

## Findings drive reflex re-planning

Any phase can `report_finding` — a vuln, bug, or design flaw with a severity.
The mission engine's replan step turns open findings at or above a configured
severity threshold into deterministic fix-and-reverify tasks, deduplicating
recurring findings and capping the total so the loop can't run away. This is
the automatic, rule-based half of re-planning.

## A lead can reshape the plan

Beyond that automatic reflex, an LLM acting as lead can actively reshape a
running mission — cancelling a task, retargeting dependencies, reopening a
task, or superseding one whose premise changed, with dependents automatically
rewritten to the replacement.

## Review gate → sprints

A mission created with review required waits for a client verdict instead of
auto-completing. The **client** role — a human, or a modeled product-owner
agent in the demo — either accepts (the mission converges) or requests
changes, which starts a new **sprint**: a fresh round of phases addressing the
feedback. Sprints are capped so a never-satisfied client can't loop forever —
the mission must be accepted or the directive revised. Nothing converges
without that acceptance step: the same human-gate principle that governs
[skill proposals](/docs/concepts/memory-and-learning-loop/) and
[trust scope](/docs/concepts/trust-and-security/) elsewhere in the system.

See `corral-admin mission list|status <id>|create <directive...>` in the
[CLI reference](/docs/cli/corral-admin/) to drive missions from the command
line, [The task queue + verify gate](/docs/concepts/queue-and-verify/) for how
individual tasks complete, and
[Mission history + replay](/docs/concepts/history-and-replay/) for how a
finished mission is recorded and replayed afterward.
