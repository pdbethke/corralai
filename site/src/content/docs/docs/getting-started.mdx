## Install

Clone and build from source:

```bash
git clone https://github.com/pdbethke/corralai
cd corralai
go build ./...
```

## Your first mission

Against a brain you're running yourself (dev mode — no `CORRALAI_OIDC_ISSUER`
set, so auth is off):

```bash
go run ./cmd/corral
```

Open `http://127.0.0.1:9019/` for the live corral view and the **Progress**
tab. From another terminal, drive it with `corral-admin` — see the
[CLI reference](/docs/cli/corral-admin/) for every verb, including
`mission list|status <id>|create <directive...>`.

## The key-free demo

The fastest way to watch a whole mission converge on one command (bundled GPU
Ollama, no API keys required):

```bash
cd deploy/demo
make demo-mission
```

This brings up the brain plus the full role team (builder, tester, pentester,
reviewer, plus a lead and a client agent), seeds a real directive, and you
watch it converge in the Progress tab: directive → team builds it →
re-plans → client review → converge. See
[deploy/demo/README.md](https://github.com/pdbethke/corralai/blob/main/deploy/demo/README.md)
for the full walkthrough.

## Common knobs

- `CORRALAI_OIDC_ISSUER` / `CORRALAI_OIDC_AUDIENCE` — cross-machine auth
- `CORRALAI_GIT_TOKEN` + `CORRALAI_FORGES` — repo-work / multi-forge missions
- `CORRALAI_EMBED_URL` — reference RAG + vector search
- `CORRALAI_MOTHERDUCK` — fleet analytics + oracle
- `MODEL_BACKEND` / `OPENAI_BASE_URL` — bring your own model

See [Running it](/docs/running-it/) for the full rundown, and
[the CLI reference](/docs/cli/corral-admin/) for every `corral-admin` verb.
