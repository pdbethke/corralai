The Topology tab is the corral's ecosystem map: every host machine, the
audit-role agents running on it, and — for each agent — the model backing it
and the exec-isolation jail confining its commands. It's observability, not
control: useful for seeing at a glance which host is carrying the load,
whether a decorrelated pair is actually on two different models, or whether
an agent is running with network access it shouldn't need.

The decorrelation the audit depends on shows up here directly: a
mutant-generator and its shadow challenger on different models, and a
test-critic that's never the same model (and often a different vendor) than
the test-writer it's reviewing behind — "a judge may not certify herself"
made visible as a topology fact, not just a claim in the docs.
